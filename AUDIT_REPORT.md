## Top critiques transverses (à traiter en priorité)

1. **SQL injection** dans `services/ecommerce/src/routes/stock/index.ts:214` (raw query avec interpolation `${query.threshold}`)
2. **SQL injection** dans `services/analytics/src/routes/sales.ts:63` (interpolation du `group` dans `$queryRaw`)
3. **Webhook Stripe** sans return early si `STRIPE_WEBHOOK_SECRET` absent — `services/ecommerce/src/routes/checkout/index.ts:246`
4. **IDOR** sur `GET /orders/:id` — `services/ecommerce/src/routes/orders/index.ts:1257` (CLIENT peut accéder à n'importe quelle commande si pas ADMIN)
5. **Cron interne CRM** sans HMAC — `services/crm/src/index.ts:176` (en-tête `x-internal-cron` spoofable, déclenche envois email/SMS)
6. **JWT en localStorage** côté front — `apps/web/src/lib/api.ts:60` (XSS-vulnerable, devrait être cookie httpOnly)
7. **Réponses analytics non conformes** au contrat `{success, data, error}` — toutes les routes `services/analytics/src/routes/*.ts`
8. **N+1 critiques** dans CRM (campaigns send loop, triggers notification idempotence) — `services/crm/src/routes/campaigns/index.ts:328` et `triggers/index.ts:205-273`
9. **Schemas Prisma** : Shipment, CampaignSend, NotificationLog sans `onDelete` ni relations correctes — `packages/database/prisma/schema.prisma`

## apps/web

### [CRITICAL] security — Hardcoded phone number in public code
`src/lib/brand.ts:94`
Phone number visible as plain text in browser bundle (NEXT_PUBLIC_BRAND_PHONE_INTL). Although marked as public, this enables direct targeting for spam/phishing and should be behind a server endpoint for authenticated access.
**Fix:** Move sensitive contact data to server-only environment variables accessed via API endpoint. Return masked version to client if needed.

### [CRITICAL] security — accessToken stored in localStorage without secure flag
`src/lib/api.ts:60`
JWT token retrieved from localStorage and sent in every API call with no secure storage. localStorage is vulnerable to XSS attacks and does not respect httpOnly flag requirements.
**Fix:** Use secure httpOnly cookies for tokens instead of localStorage. Implement CSRF tokens for state-changing operations.

### [CRITICAL] perf — N+1 product fetches on category filter
`src/app/(shop)/produits/CatalogueFilters.tsx`
Each product card triggers a separate fetch when filter changes. Should batch or use single endpoint with filtering applied server-side.
**Fix:** Move filtering logic to server component. Use URL search params with single paginated API call including filters.

### [CRITICAL] next-best-practices — Missing error boundary for async pages
`src/app/(shop)/produits/[slug]/page.tsx:91-144`
Server component page with no error.tsx in route group. If API call fails, users see default Next.js 500 page instead of branded error.
**Fix:** Create `src/app/(shop)/produits/[slug]/error.tsx` with proper error UI.

### [HIGH] security — Unvalidated URL parameter in redirect
`src/app/(shop)/checkout/page.tsx:55`
`return_url` set to `${window.location.origin}/mon-compte?commande=${orderId}` with orderId from Stripe response. If orderId is user-controlled, open redirect possible.
**Fix:** Validate orderId is UUID format before constructing URL. Use Next.js `useRouter().push()` instead of direct URL manipulation.

### [HIGH] security — Missing CSRF protection on guest checkout
`src/app/(shop)/checkout/page.tsx:284`
Guest checkout sends POST to `/api/v1/orders/guest` with only `x-session-id` header. Session ID is client-generated random UUID, not cryptographically bound to user.
**Fix:** Add CSRF token header generated by server. Validate session ID matches secure cookie.

### [HIGH] security — Exposed `any` type casting in admin edit
`src/app/(admin)/admin/produits/[id]/page.tsx:128`
Line 128: `setBrandId((p.brand as any)?.id || "")` defeats type safety. Could silently accept malformed brand data.
**Fix:** Define explicit Brand interface and use safe optional chaining: `setBrandId(p.brand?.id ?? "")`.

### [HIGH] perf — Unoptimized Image component without priority
`src/app/(shop)/produits/[slug]/page.tsx:48-53`
ProductGallery renders multiple images without `priority` flag. First image should use `priority` to avoid layout shift and improve LCP.
**Fix:** Add `priority={i === 0}` to Image component in ProductGallery.

### [HIGH] perf — Missing Suspense boundary for related products
`src/app/(shop)/produits/[slug]/page.tsx:154`
Related products fetch blocks rendering. Page waits for 4 related product cards before showing main content.
**Fix:** Wrap related products section in `<Suspense>` with fallback skeleton.

### [HIGH] a11y — Missing alt text on key images
`src/app/(shop)/produits/[slug]/ProductGallery.tsx:48`
Line 48: `alt=""` on thumbnail gallery images. Should describe which image number or visual change it shows.
**Fix:** Change to descriptive alt like `View image ${i + 1}`.

### [HIGH] techdebt — 8 `as any` casts across admin pages
`src/app/(admin)/admin/produits/[id]/page.tsx:128,134,137,140,147`
Unsafe type casts bypass TS strict mode and risk runtime errors.
**Fix:** Define proper DTO interfaces for API responses. Use type guards or Zod validation.

### [MEDIUM] bug — Race condition on password reset token validation
`src/app/(shop)/reset-password/page.tsx:49`
Token from URL is used immediately in API call without debouncing. Multiple rapid navigation events could trigger duplicate reset attempts.
**Fix:** Add `useTransition` and prevent multiple submissions.

### [MEDIUM] security — Unescaped HTML in product descriptions
`src/app/(shop)/produits/[slug]/page.tsx:368,376`
Uses `sanitizeProductHtml()` which allows certain tags. No CSP header configured. If sanitizer is misconfigured, XSS risk exists.
**Fix:** Add CSP header in next.config.ts. Verify sanitize-html whitelist (currently allows `<a href>` which could lead to javascript: URLs).

### [MEDIUM] perf — Full page re-render on theme switch
`src/lib/themes.ts`
Theme change via `localStorage.setItem()` triggers full document repaint.
**Fix:** Use CSS variables that update via `data-theme` attribute without re-rendering React tree.

### [MEDIUM] techdebt — Dead code in formatDate utility
`src/app/(shop)/mon-compte/suivi/[token]/page.tsx:20-26`
`formatDate` function defined locally but same logic exists in `lib/utils.ts`. Violates DRY.
**Fix:** Import `formatDate` from shared utility library.

### [MEDIUM] bug — Unhandled promise rejection in ProductReviews
`src/app/(shop)/produits/[slug]/ProductReviews.tsx:21`
`.catch(() => setLoaded(true))` suppresses error silently.
**Fix:** Store error state and display message.

### [MEDIUM] a11y — Color contrast issue in spec-label
`src/app/(shop)/produits/[slug]/page.tsx:249`
Spec label uses `text-neon` color. Against dark background, neon cyan (#00FFD1) may fail WCAG AA contrast ratio.
**Fix:** Verify with WCAG Contrast Checker. Consider darker shade.

### [LOW] next-best-practices — Missing generateStaticParams revalidation
`src/app/(shop)/guide/[slug]/page.tsx:162`
`generateStaticParams()` hardcodes all guide slugs. If guides added at runtime, ISR should be enabled.
**Fix:** Add `export const revalidate = 3600`.

### [LOW] techdebt — Duplicated price formatting logic
`src/app/(shop)/produits/[slug]/page.tsx:29-35`
`formatHT()` duplicates Intl.NumberFormat logic from `lib/utils.ts`.
**Fix:** Use existing utility `formatPrice()` from `lib/utils.ts`.

### [LOW] perf — Unnecessary re-renders in AddToCartSection
`src/app/(shop)/produits/[slug]/AddToCartSection.tsx:22-39`
`handleAddToCart` is recreated on every render.
**Fix:** Wrap with `useCallback`.

### [LOW] a11y — Missing label associations in cart quantity input
`src/app/(shop)/panier/page.tsx`
Quantity input likely has no associated `<label>` with matching `htmlFor`.
**Fix:** Wrap input in `<label>` or add `aria-label="Quantity"`.

### [LOW] security — Unvalidated environment variable fallback
`src/lib/api.ts:10`
`process.env.NEXT_PUBLIC_API_ECOMMERCE || 'http://localhost:3001'` falls back to localhost in production if env var missing.
**Fix:** Throw at build time if missing.

## services/ecommerce

### [CRITICAL] security — IDOR on GET /orders/:id
`src/routes/orders/index.ts:1257`
Endpoint checks if user is ADMIN but allows CLIENT role to access any order if not admin. Should enforce both RBAC and ownership check.

### [CRITICAL] security — Unsafe Stripe webhook signature verification
`src/routes/checkout/index.ts:246-251`
The code checks if webhook secret is set but does not throw immediately if missing, allowing webhook processing to continue without proper verification.

`src/routes/stock/index.ts:214`
Raw SQL concatenates `query.threshold` directly into the query: `stock_quantity <= ${query.threshold ?? 0}`.

`src/routes/reviews/index.ts:33`
Query parameters cast as `request.query as { page?: string; limit?: string }` without Zod validation.

Endpoint denies CLIENT but allows STAFF (lower privilege). Should restrict to MANAGER+.

### [HIGH] perf — N+1 in order cancel stock release

### [HIGH] perf — Missing pagination on GET /admin/users

### [HIGH] perf — Missing select on full order includes

### [MEDIUM] techdebt — Type cast without validation in admin-users
`src/routes/admin-users/index.ts:55`
Query parameters cast as `{ role?: string; status?: string }` without Zod validation.
**Fix:** Add Zod schema to validate role and status enums.

### [MEDIUM] techdebt — Type cast in reviews route
`src/routes/reviews/index.ts:85`
Route params cast as `{ slug: string }` without validation.

### [MEDIUM] techdebt — Type cast in admin products delete
`src/routes/admin/index.ts:313`

### [MEDIUM] techdebt — String-based error matching in stock

Stock movement error handling uses `.catch()` to check `err.message` string instead of typed error classes.

### [MEDIUM] bug — Race condition in installment order status transitions

Stock release uses `paymentMethod.startsWith("INSTALLMENT_")` without verifying installment status.

### [MEDIUM] api-contract — Inconsistent guest order checkout response
`src/routes/orders/index.ts:884`
Guest checkout returns 201 with `data: fullOrder` but guest user email may not be captured. Response shape differs from authenticated checkout.

### [MEDIUM] perf — Inefficient cart product enrichment
`src/routes/cart/index.ts:96-112`
For each variant ID, separate `productVariant.findMany()` is executed.

### [MEDIUM] security — Insufficient rate limit on guest checkout

### [MEDIUM] bug — Missing null check before parsing JSON cart data
`src/routes/checkout/index.ts:150`

### [LOW] techdebt — Placeholder invoice details hardcoded

## services/crm

### [CRITICAL] security — Unauthenticated internal cron endpoint
`src/index.ts:176-191`
Trigger execution endpoint checks only `x-internal-cron` header which can be spoofed. Allows attackers to trigger expensive operations (email/SMS campaigns) repeatedly.
**Fix:** Use HMAC signature or IP allowlist instead of simple header check.

### [CRITICAL] bug — N+1 in campaign send loop
`src/routes/campaigns/index.ts:328-357`
Loop performs `findUnique` query per profile to check idempotence then a separate `create`. With thousands of recipients, generates 2N queries.
**Fix:** Batch idempotence check with `findMany({where: {customerId: {in: profiles.map(p => p.userId)}}})` and use a Map.

### [CRITICAL] bug — N+1 in trigger execution notification checks
`src/routes/triggers/index.ts:205-216, 235-246, 263-273`
Each ticket triggers a `findFirst` to check if notification already sent. With 1000+ tickets, becomes 1000+ sequential DB hits.
**Fix:** Batch fetch with `findMany({where: {ticketId: {in: tickets.map(t => t.id)}}})` and use a Set.

### [CRITICAL] techdebt — Type-unsafe email validation
`src/routes/campaigns/index.ts:230`
Email input uses `request.body as { email?: string }` without schema validation.
**Fix:** Use Zod: `z.object({email: z.string().email()}).parse(request.body)`.

### [HIGH] security — Inconsistent RBAC across trigger routes
`src/routes/triggers/index.ts:62`
POST /triggers checks `!== "CLIENT" && !== "TECHNICIAN"` but allows STAFF; GET /triggers (line 43) excludes STAFF. STAFF can create triggers but not list them.
**Fix:** Define `MANAGER_ROLES = ['SUPERADMIN', 'ADMIN', 'MANAGER']` and use consistently.

### [HIGH] perf — Missing pagination on segments list
`src/routes/segments/index.ts:65-71`
GET /segments returns all without pagination.
**Fix:** Add page/limit Zod schema with `skip/take`.

### [HIGH] perf — Missing pagination on campaigns list
`src/routes/campaigns/index.ts:73-78`
GET /campaigns returns all campaigns without limit.
**Fix:** Add pagination matching customers endpoint pattern.

### [HIGH] bug — Race condition: loyalty points outside transaction in customer merge
`src/routes/customers/index.ts:709-724`
Loyalty points merge executes outside main transaction.
**Fix:** Move loyalty points update inside the `$transaction` array.

### [HIGH] bug — Missing loyaltyLog migration during customer merge
`src/routes/customers/index.ts:663-707`
Customer merge transfers orders, tickets, addresses, interactions, reviews — but omits loyaltyLog/loyaltyPoint records.
**Fix:** Add `loyaltyPoint.updateMany` to the transaction.

### [MEDIUM] security — Unsafe dynamic import with type assertion
`src/routes/triggers/index.ts:338`
`await import("nodemailer" as string)` bypasses type checking.
**Fix:** `await import("nodemailer")` without assertion.

### [MEDIUM] api-contract — POST returns 201 instead of 200
`src/routes/campaigns/index.ts:122`, `src/routes/triggers/index.ts:82`
Inconsistent HTTP code with rest of monorepo.
**Fix:** Standardize on 200 or document the convention in CLAUDE.md.

### [MEDIUM] techdebt — Inconsistent RBAC error responses in triggers
`src/routes/triggers/index.ts:43,62,92,122,150`
French strings ("Acces reserve aux managers") instead of using `ForbiddenError` from @trottistore/shared.
**Fix:** Throw `new ForbiddenError(...)` consistently.

### [MEDIUM] bug — Segment date calculation timezone-unsafe
`src/routes/segments/index.ts:52-56`
`new Date()` is local time and `setDate()` doesn't account for timezone.
**Fix:** Use UTC: `new Date(Date.now() - days * 86400000)`.

### [MEDIUM] techdebt — Duplicate `buildProfileWhere` across segments and campaigns
`src/routes/segments/index.ts:25`, `src/routes/campaigns/index.ts:47`
Same logic defined twice.
**Fix:** Extract to shared utility `utils/segmentation.ts`.

### [MEDIUM] api-contract — Missing pre-transaction validation in customer merge
`src/routes/customers/index.ts:637-641`
Merge endpoint validates UUID format but does partial writes if second lookup fails.
**Fix:** Move existence checks before transaction starts.

### [LOW] techdebt — Unused parameters in campaign routes
`src/routes/campaigns/index.ts:73`
GET /campaigns handler declares no parameters; inconsistent with other handlers.
**Fix:** Use `(_request, _reply)` for consistency.

### [LOW] perf — Stale segment count after creation
`src/routes/segments/index.ts:79-91`
Segment count calculated once at creation; becomes stale.
**Fix:** Recalculate on demand or via cron.

### [LOW] bug — Invalid email format not rejected in campaign preview
`src/routes/campaigns/index.ts:254`
`sendEmail()` called with user-supplied email without validation.

`src/routes/sales.ts:63`
`group` interpolated into raw SQL via template string. Validated as enum but `$queryRaw` template-string vars are inserted as raw SQL.

`src/routes/events.ts:59`
POST /analytics/events returns `{ accepted, message }` instead of `{ success, data, error? }`.

### [HIGH] security — Type-unsafe properties record
`src/routes/events.ts:17`
Event schema accepts `properties: z.record(z.unknown()).optional()` allowing arbitrary nested structures.

### [HIGH] perf — Heavy aggregations on transactional DB
`src/routes/kpis.ts:104`
KPI endpoints fall back to raw SQL `GROUP BY` + `HAVING COUNT(*) > 1` on `ecommerce.orders` when projections empty. Full-table scans.

### [HIGH] perf — Hidden N+1 risk in cockpit
`src/routes/cockpit.ts:134`
`customerInteraction.findMany` with nested customer select. Combined with 6 other parallel queries, response time unmonitored.

### [HIGH] bug — Timezone-unsafe date construction
`src/routes/realtime.ts:26`, `src/routes/cockpit.ts:14`
`new Date(now.getFullYear(), now.getMonth(), now.getDate())` uses local TZ. KPIs offset by hours in non-UTC servers.

### [HIGH] bug — Incorrect conversionRate calculation
`src/routes/kpis.ts:127`
`(ordersCount / (ordersCount * 25 || 1)) * 100` always equals 4%. Hardcoded mock visitor estimate.
**Fix:** Remove mock or fetch actual visitor data from analytics events/ClickHouse.

### [MEDIUM] api-contract — Missing response wrapping (6 GET endpoints)
`src/routes/sales.ts:94`
All GET analytics endpoints return raw objects.

### [MEDIUM] perf — Top customers query without NULLS LAST

`ORDER BY cp.total_spent DESC` without NULLS handling. Hardcoded `LIMIT 5`.

### [MEDIUM] techdebt — Duplicate KPI query logic

`runLegacyKpiQueries()` and inline returning customers query duplicated.

### [MEDIUM] perf — Inefficient `redis.keys('cart:*')` for cart count
`src/routes/realtime.ts:69`
Loads all cart keys into memory.

### [MEDIUM] bug — Cache invalidation race condition
`src/routes/kpis.ts:146`
Two concurrent requests both compute and set cache.

### [MEDIUM] techdebt — Hardcoded currency

### [LOW] perf — Unused SELECT columns in cockpit
